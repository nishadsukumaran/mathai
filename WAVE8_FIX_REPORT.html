P0 · PROFILE-03a
ProfileModal — confidence slider scale conversion
apps/web/components/mathai/profile-modal.tsx
✓ Done
The DB stores confidence on a 0–100 EWMA scale. The API PATCH endpoint expects 1–5 (Zod validated). The ProfileModal was passing the raw 0–100 DB value directly to useState, so the slider appeared maxed out and any save was rejected with a 400 (Zod min/max violation), causing the modal's save to silently fail for all real users. ProfilePageContent already had this conversion — now ProfileModal has it too. 
- const [confidence, setConfidence] = useState<number>(data.confidenceLevel);
+ const rawConf = data.confidenceLevel ?? 60;
+ const [confidence, setConfidence] = useState<number>(
+   Math.max(1, Math.min(5, Math.round(rawConf / 20))),
+ );
P0 · API-01
clientApi — 401 now redirects to sign-in
apps/web/lib/clientApi.ts
✓ Done
When the NextAuth session expired, /api/auth/token returned 401. getToken() fell through to "Bearer dev-stub", which the Express API rejected. All client-side saves (profile, practice) failed silently with null returns. This was also the root cause of grade edits not being reflected — clientPatch returned null so updateSession was never called. Now a 401 from the token endpoint triggers a redirect to sign-in instead. 
// Before: fell through to dev-stub on any non-ok response
- if (res.ok) { const { token } = ...; return `Bearer ${token}`; }
- return "Bearer dev-stub";

// After: explicit 401 handling
+ if (res.ok) { const { token } = ...; return `Bearer ${token}`; }
+ if (res.status === 401) { window.location.href = "/auth/signin"; return ""; }
+ // Only dev-stub on network failure (offline)
+ return "Bearer dev-stub";

🔎 Root Cause — "Grade Edit Not Reflecting"

The grade edit flow in ProfilePageContent was architecturally correct: save → updateSession({ grade }) → JWT refreshed. The real blocker was a 3-step chain:
  1. Session expires → /api/auth/token returns 401
  2. getToken() falls to "Bearer dev-stub" → Express rejects PATCH with 401/403
  3. clientPatch returns null → if (updated) is false → updateSession never called → grade not stored
Fixed by API-01: getToken() now redirects to sign-in on 401 instead of silently degrading. Once the user re-authenticates, the grade save + session sync works end-to-end.
