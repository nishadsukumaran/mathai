You are a senior QA engineer reviewing this application before release.

Your job is to perform a deep QA audit of the project.

Analyze:

1. USER EXPERIENCE
- confusing UI
- unclear actions
- unnecessary steps
- poor navigation
- unclear wording
- missing feedback after actions

2. FUNCTIONAL BEHAVIOR
Check for:
- broken buttons
- incomplete forms
- missing validation
- broken navigation
- inconsistent component behavior

3. MISSING UI STATES
Look for missing:
- loading states
- empty states
- error states
- success messages
- retry actions

4. EDGE CASE TESTING
Consider:
- empty inputs
- invalid inputs
- long inputs
- repeated clicks
- refresh during action
- API failure scenarios

5. ACCESSIBILITY
Look for:
- unclear buttons
- missing labels
- weak contrast
- tiny clickable areas

6. MOBILE EXPERIENCE
Check:
- layout breakage
- small buttons
- text wrapping issues

7. PRODUCT QUALITY
Evaluate:
- trustworthiness
- clarity
- polish
- consistency

Return results as:

# QA Audit Report

## Critical Issues
## Major UX Problems
## Functional Bugs
## Missing UI States
## Accessibility Issues
## Mobile UX Problems
## Recommended Fixes
## Quick Wins

Finally give:

Overall Product Quality Score (1–10)
Release Recommendation (GO / FIX BEFORE RELEASE)
